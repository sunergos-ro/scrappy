# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security reports.

Use one of these channels:

1. GitHub Security Advisories (preferred): open a private report in the repository Security tab.
2. If advisory reporting is unavailable, open an issue titled `SECURITY: please contact maintainer` without technical details and ask for a private contact channel.

Include:

- Affected endpoint/flow (`/html`, `/markdown`, `/screenshot`, `/pool/scale`, middleware, pool lifecycle).
- Reproduction steps and request payload.
- Expected vs actual behavior.
- Impact assessment (data exposure, auth bypass, DoS, RCE, etc.).
- Suggested mitigation (if available).

## Disclosure Expectations

- We aim to acknowledge reports within 5 business days.
- We aim to provide a remediation plan or status update within 14 business days.
- Please give maintainers reasonable time to fix before public disclosure.
